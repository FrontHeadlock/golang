# 한국어 Skill 문서 작성

## 개요

- `.codex/skills` 아래 skill 문서 작성 전용 규칙
- 적용 시점, 강제 규칙, 기대 결과의 즉시 파악 가능 형태 지향

## 적용 조건

- `.codex/skills` 아래 새 skill 생성 시점
- 기존 skill 규칙 또는 템플릿 수정 시점
- 반복 가능한 작업 절차 문서화 시점

## 언어 규칙

- 설명, 규칙, 예시, 로그성 텍스트 = 모두 한국어
- 코드 예시 내 코드 토큰 = 원문 유지
- 코드 블록 내부 주석 = 필요 시 한영 병기 허용
- 코드 블록 외부 설명 = 한국어만 사용
- 예약어, 명령어, 식별자, frontmatter 키 = 원문 유지 가능
- 불필요한 영어 설명 문장 사용 금지
- 문장 마무리 = 명사형 우선
- 장문 설명보다 개조식 우선

## 작성 원칙

- 적용 시점의 한눈 파악 가능 구조
- 모호성 없는 규칙 표현
- 출력 형식 또는 템플릿 필수 포함
- 학습용 프로젝트 기준의 실제 사용 가능 규칙만 유지
- 하나의 스킬 = 하나의 주된 목적

## 출력

- 자연스러운 한국어 설명
- 한국어 문서 문자열
- 한국어 중심의 Markdown 스킬 문서

## 작성 절차

1. 사용 시점 정의
2. 적용 조건 및 언어 규칙 정리
3. 핵심 규칙 및 출력 형식 작성
4. 필요 시 절차와 템플릿 추가
5. 체크리스트와 금지 사항 정리

## Skill 템플릿

````md
---
name: skill-name
description: Use when ...
---

# 스킬 제목

## 개요
- 이 스킬의 목적 정리

## 적용 조건
- 적용 시점 정리

## 규칙
- 반드시 지켜야 할 규칙 1
- 반드시 지켜야 할 규칙 2

## 출력
- 기대하는 출력 형식 1
- 기대하는 출력 형식 2

## 절차
1. 먼저 할 일
2. 다음 할 일
3. 완료 기준

## 템플릿
```md
# 필요 시 실제 출력 템플릿 정리
```

## 체크리스트
- 빠진 규칙 유무 확인
- 설명의 한국어 유지 확인
````

## 작성 체크리스트

- 스킬 적용 시점의 분명성
- 규칙의 모호성 여부
- 출력 형식의 재사용 가능성
- 코드 바깥 설명의 한국어 유지
- 실제 프로젝트 관행과의 충돌 여부
- 다른 스킬과의 역할 중복 여부

## 금지 사항

- 영어 문단 기반 규칙 설명 금지
- 트리거 없는 스킬 목적만의 단독 기재 금지
- 결과 형식 없는 막연한 지침 나열 금지
- 한 스킬 내 서로 다른 목적의 과도한 혼합 금지
