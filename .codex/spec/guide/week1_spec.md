# Go 8주 집중 학습 로드맵 - Week 1 학습 가이드형 Spec

## 1. 문서 목적

이 문서는 `프로그래밍 경험은 있지만 Go는 처음인 학습자`를 위한 1주차 학습 가이드다.
목적은 문법 목록을 보여주는 것이 아니라, Go를 처음 접하는 학습자가 어떤 순서로 사고를 전환하고 어떤 수준까지 손에 익혀야 하는지 명확히 안내하는 데 있다.

이 스펙은 아래 질문에 답해야 한다.

- 이번 주에 무엇을 이해해야 하는가
- 어떤 개념이 Go 초심자에게 특히 낯선가
- 각 날짜가 끝났을 때 무엇을 설명하고 무엇을 직접 구현할 수 있어야 하는가

## 2. 대상 학습자

이 스펙은 아래 학습자를 기준으로 한다.

- 기본적인 프로그래밍 개념은 이미 익숙하다.
- Java, Python, JavaScript 같은 언어를 최소 하나 이상 사용해 본 적이 있다.
- CLI 실행, 파일 읽기, JSON 구조는 크게 낯설지 않다.
- Go는 처음이거나, 튜토리얼 수준으로만 접해 봤다.
- 장기적으로는 시스템 도구, 자동화 도구, 백엔드 서비스, CI/CD 관련 프로그램을 Go로 구현하고 싶다.

## 3. 1주차 학습 목표

1주차가 끝나면 아래 상태에 도달해야 한다.

- Go 프로그램의 기본 실행 구조와 프로젝트 시작 방식을 설명할 수 있다.
- 변수, 함수, 제어문, slice, map, struct, method, interface, error 처리의 기본 문법을 직접 작성할 수 있다.
- Java의 class 중심 사고나 Python의 exception 중심 사고를 그대로 들고 오지 않고 Go식 구조를 받아들일 수 있다.
- JSON 파일을 읽고 검증해서 출력하는 작은 CLI 프로그램을 혼자 조립할 수 있다.
- 이후 2주차에서 package 설계, 테스트, 프로세스 실행 같은 확장 주제로 넘어갈 준비가 된다.

## 4. 1주차 핵심 학습 관점

### 4.1 Go는 class 중심 언어가 아니다

Go에서는 `class -> inheritance`보다 `struct -> method -> interface`가 기본 구조다.
따라서 "객체를 상속해서 기능을 늘린다"보다 "데이터를 struct로 정의하고 필요한 동작을 메서드로 붙인다"는 관점으로 접근해야 한다.

### 4.2 Go의 error는 제어 흐름의 일부다

Go는 예외를 던져 흐름을 끊기보다, error를 반환값으로 드러내는 방식을 선호한다.
1주차에는 `if err != nil`을 외워서 쓰는 것이 아니라, 왜 Go가 이 방식을 선택했는지 이해하는 것이 중요하다.

### 4.3 slice는 실무 기본 자료구조다

배열을 배우더라도 실제로 자주 다루게 되는 것은 slice다.
따라서 1주차 자료구조 학습의 중심은 array 자체가 아니라 slice의 감각을 익히는 데 둔다.

### 4.4 pointer는 메모리보다 상태 변경 관점으로 먼저 이해한다

초반에는 포인터를 저수준 메모리 주소 조작으로 이해하려고 하면 진입 장벽이 높아진다.
먼저 아래 질문으로 이해한다.

- 원본 값을 바꿔야 하는가
- 값 복사를 피하는 편이 더 자연스러운가
- 메서드가 상태를 수정하는 책임을 가지는가

### 4.5 Go는 단순하지만 엄격하다

Go는 문법이 화려하지 않지만 아래 규칙을 강하게 요구한다.

- 사용하지 않는 변수는 허용하지 않는다.
- 사용하지 않는 import는 허용하지 않는다.
- 숨겨진 동작보다 명시적 흐름을 선호한다.
- 짧고 단순하며 읽기 쉬운 코드를 권장한다.

즉, 1주차는 "문법을 많이 외운다"가 아니라 "Go가 어떤 스타일을 강제하는지 몸으로 익힌다"가 핵심이다.

## 6. 일별 학습 가이드

### Day 1 - Go 실행 구조와 기본 문법

**오늘의 목표**

- Go 프로그램이 어떻게 시작되고 실행되는지 이해한다.
- 변수 선언 방식과 zero value 개념을 익힌다.

**오늘 반드시 이해할 개념**

- `package main`
- `func main()`
- `import`
- `var`, `:=`, `const`
- 기본 타입: `int`, `string`, `bool`, `float64`
- `fmt.Println`, `fmt.Printf`
- zero value
- unused variable 에러

**오늘 해야 할 일**

- `hello.go`를 빈 파일에서 직접 작성해 실행한다.
- `var`와 `:=`를 각각 사용한 예제를 만든다.
- 초기화하지 않은 변수의 zero value를 출력한다.
- 사용하지 않는 변수를 일부러 만들어 컴파일 에러를 확인한다.

**오늘 끝나면 할 수 있어야 하는 것**

- `var`와 `:=`의 차이를 설명할 수 있다.
- zero value가 왜 필요한지 말할 수 있다.
- 가장 작은 Go 실행 파일을 직접 만들 수 있다.

**자기 점검 질문**

- Go는 왜 사용하지 않는 변수를 허용하지 않을까
- `const`와 `var`는 어떤 기준으로 나눠 써야 할까

### Day 2 - 함수, 제어문, 에러 반환 패턴

**오늘의 목표**

- 함수 선언과 다중 반환값에 익숙해진다.
- `if err != nil` 패턴을 읽고 직접 쓸 수 있다.

**오늘 반드시 이해할 개념**

- 함수 선언
- 다중 반환값
- `if`, `else`
- `for`
- `switch`
- scope
- `error` 반환 패턴

**오늘 해야 할 일**

- 두 값을 반환하는 함수를 작성한다.
- 0으로 나누는 경우 에러를 반환하는 함수를 만든다.
- `switch` 분기 예제를 작성한다.
- 간단한 계산기 CLI를 구현한다.

**오늘 끝나면 할 수 있어야 하는 것**

- 여러 값을 반환하는 함수를 자연스럽게 작성할 수 있다.
- `if err != nil`이 왜 Go에서 기본 패턴인지 설명할 수 있다.
- Go에서 반복문이 `for` 하나로 통합되어 있다는 점을 이해한다.

**자기 점검 질문**

- Python의 exception과 Go error 반환은 제어 흐름 면에서 무엇이 다른가
- if-else 연쇄보다 switch가 더 읽기 좋은 경우는 언제인가

### Day 3 - 자료구조: array, slice, map, string

**오늘의 목표**

- Go에서 자주 쓰는 자료구조를 직접 다뤄 본다.
- slice와 map을 중심으로 데이터 처리 감각을 익힌다.

**오늘 반드시 이해할 개념**

- array
- slice
- `len`, `cap`
- `append`
- `range`
- map
- string / rune / byte

**오늘 해야 할 일**

- slice에 값을 추가하면서 `len`과 `cap` 변화를 관찰한다.
- `map[string]int` 기반 단어 카운터를 만든다.
- 문자열을 byte와 rune 단위로 순회해 차이를 비교한다.
- 한글 문자열 길이를 `len`과 rune 기준으로 각각 출력한다.

**오늘 끝나면 할 수 있어야 하는 것**

- array와 slice 차이를 설명할 수 있다.
- slice가 실무에서 더 자주 쓰이는 이유를 말할 수 있다.
- Unicode 문자열 처리에서 rune이 필요한 이유를 이해한다.

**자기 점검 질문**

- slice는 배열과 어떤 관계를 가지는가
- 문자열 길이를 셀 때 왜 byte와 rune을 구분해야 하는가

### Day 4 - struct, method, pointer

**오늘의 목표**

- Go에서 데이터를 struct로 모델링하고 동작을 method로 붙이는 방식을 익힌다.
- pointer receiver가 필요한 상황을 이해한다.

**오늘 반드시 이해할 개념**

- struct 선언 및 초기화
- field 접근
- method
- value receiver
- pointer receiver
- `&`, `*`

**오늘 해야 할 일**

- `Task` struct를 정의한다.
- 상태를 변경하는 메서드를 작성한다.
- value receiver와 pointer receiver를 각각 사용해 차이를 확인한다.
- 여러 `Task`를 slice로 관리하는 코드를 작성한다.

**오늘 끝나면 할 수 있어야 하는 것**

- Go에서 struct가 class와 어떻게 다른지 설명할 수 있다.
- pointer receiver가 필요한 상황을 말할 수 있다.
- 데이터를 struct와 method 조합으로 구성할 수 있다.

**자기 점검 질문**

- 객체지향 경험자가 Go struct를 처음 볼 때 가장 먼저 버려야 할 습관은 무엇인가
- receiver를 값으로 받을지 포인터로 받을지 어떤 기준으로 결정할까

### Day 5 - interface, error, defer

**오늘의 목표**

- Go의 추상화와 에러 처리 스타일에 적응한다.
- interface와 error를 예제 수준에서 직접 사용한다.

**오늘 반드시 이해할 개념**

- interface 선언
- 암묵적 구현
- `error`
- `errors.New`
- `fmt.Errorf`
- `defer`
- `panic/recover`

**오늘 해야 할 일**

- `Runner` interface를 정의한다.
- `LocalRunner` 구현체를 만든다.
- 입력 검증 함수를 작성한다.
- 에러를 생성하고 감싸는 예제를 만든다.
- 자원 정리 코드에 `defer`를 적용한다.

**오늘 끝나면 할 수 있어야 하는 것**

- Go interface가 구현 선언 없이 만족된다는 점을 설명할 수 있다.
- error를 반환하고 상위 호출부에서 처리하는 흐름을 구현할 수 있다.
- `defer`가 자원 정리에 왜 유용한지 말할 수 있다.

**자기 점검 질문**

- Java interface와 Go interface는 설계 감각이 어떻게 다른가
- panic은 언제 써야 하고, 언제 쓰면 안 되는가

**오늘의 목표**

## 7. 주간 학습 원칙

- 예제 코드는 복붙하지 않고 직접 타이핑한다.
- 매일 최소 1개 코드는 빈 파일에서 다시 작성한다.
- 개념을 확인한 직후 바로 손으로 구현한다.
- 코드를 한 번 실행하는 것보다, 왜 그렇게 작성했는지 설명할 수 있는 상태를 더 중요하게 본다.
- 매일 `오늘 배운 점`, `헷갈린 점`, `내일 다시 볼 점`을 짧게라도 기록한다.

## 8. 권장 학습 시간

- 평일: 2.5 ~ 3시간
- 주말: 4 ~ 5시간

## 9. 학습 완료 기준

이 문서 기준에서 1주차 완료는 아래 조건을 만족할 때 인정한다.

- Go 실행 구조와 package 구조를 설명할 수 있다.
- 변수, 함수, slice, map, struct, interface, error의 기본 문법을 직접 작성할 수 있다.
- `struct + method`, `error 반환`, `slice 중심 자료구조`, `package 분리`를 Go다운 기본 방식으로 받아들인다.
- JSON 파일을 읽고 struct로 파싱하는 과정을 구현할 수 있다.
- 작은 CLI 프로그램을 혼자 구성할 수 있다.

## 10. 1주차 종료 시 기대 상태

1주차가 끝나면 아래 상태여야 한다.

- Go 코드를 읽을 때 문법 낯섦 때문에 멈추는 빈도가 크게 줄어든다.
- Java/Python식 구조를 억지로 가져오지 않고 Go식 코드를 읽고 쓰기 시작한다.
- 파일 입력 기반 CLI 정도는 혼자 구현할 수 있다.
- 이후 package 설계, 테스트, 프로세스 실행 등 확장 학습으로 넘어갈 준비가 된다.
